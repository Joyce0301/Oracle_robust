# Protocol Invariants

> **什么是 Invariant？**  
> 不变量是协议在任何时刻、任何条件下都必须保持为真的陈述。  
> 如果一个 invariant 被违反 → 系统进入未定义状态 → 必须立即停止或触发断路器。

---

## 📌 符号说明

- `HF(u)`: 用户 u 的健康因子 (Health Factor)
- `collateral(u)`: 用户 u 的抵押品价值（按预言机价格折算）
- `debt(u)`: 用户 u 的债务价值
- `LTV`: 最大贷款价值比 (Loan-to-Value)
- `liquidationThreshold`: 清算阈值（通常 > LTV）

**标准关系：**
```
HF = collateral * liquidationThreshold / debt
HF < 1 → 可清算
HF ≥ 1 → 不可清算
```

## 🔒 Core Invariants

### I-01: 价格非负性
```
∀ asset, ∀ time: getPrice(asset) >= 0
```
**违反后果**：债务计算错误、清算逻辑反转  
**检查点**：每次 `getPrice` 返回后  
**测试对应**：T-01, T-04

---

### I-02: 同一区块内价格一致性
```
∀ asset, ∀ block: 同一区块内多次调用 getPrice(asset) 返回相同值
```
**违反后果**：闪电贷可利用单区块内价格变化套利  
**检查点**：预言机接口  
**测试对应**：T-01, T-09

---

### I-03: 清算条件完备性
```
∀ user: HF(user) < 1 → 该用户必须可被清算
```
**违反后果**：坏账累积、协议资不抵债  
**检查点**：每次价格更新后，扫描可清算仓位  
**测试对应**：T-01, T-04, T-06

---

### I-04: 健康用户保护
```
∀ user: HF(user) >= 1 → 该用户不应被清算
```
**违反后果**：错误清算、用户资金损失  
**检查点**：清算触发前  
**测试对应**：T-01（价格操纵导致错误清算）

---

### I-05: 协议偿付能力
```
总抵押品价值（按清算价格折算） >= 总债务价值
```
**违反后果**：协议破产、存款人无法提款  
**检查点**：每次清算后  
**测试对应**：T-04（清算级联）、T-06

---

### I-06: 预言机新鲜度
```
当前区块时间戳 - lastPriceUpdateTimestamp <= maxAge
```
**违反后果**：使用陈旧价格导致错误清算  
**检查点**：每次 `getPrice` 调用  
**测试对应**：T-02, T-05

---

### I-07: 价格变化率上限（如适用）
```
|price(t) - price(t-1)| / price(t-1) <= maxChangePercent （同一区块内）
```
**违反后果**：闪电贷瞬时操纵  
**检查点**：价格更新时  
**测试对应**：T-01, T-09

---

## 🔄 TWAP-Specific Invariants

### I-08: TWAP 单调性约束
```
TWAP 窗口内，累计价格变化有上限
```
**违反后果**：TWAP 被极端值污染  
**检查点**：TWAP 计算时  
**测试对应**：T-04, T-07

---

### I-09: 窗口完整性
```
TWAP 窗口长度 >= 最小有效窗口（如 30 分钟）
```
**违反后果**：窗口太短 → 近似现货 → 易被操纵  
**检查点**：预言机初始化/配置时  
**测试对应**：T-01, T-07

---

## 🚨 Circuit Breaker Invariants

### I-10: 异常状态停止服务
```
当任何核心 invariant 被违反时，协议必须进入暂停/冻结状态
```
**违反后果**：协议在未知状态下继续运行  
**检查点**：每次状态变更后  
**测试对应**：T-03, T-05, T-08

---

### I-11: 恢复需要人工干预
```
协议从暂停状态恢复必须经过 timelock + 多签
```
**违反后果**：攻击者可反复触发暂停/恢复套利  
**检查点**：管理函数  
**测试对应**：T-08
